## 聞いた発表

### 1. 杉本礼彦さん（痛快技術株式会社 代表取締役）— 「自動化じゃなくて自律化」

一番刺さった。技術の話ではなく、**期待値と勝ち筋**の話。

#### スライド：「杉本なりのPhysical AIの階層」

見出しがこれ。

> **現場に近い層で勝負できるよね。赤枠は勝負できるでしょ？赤枠でもすげー金うごきそうよね。**

図の構造（上から下へ）：

| レイヤー | 中身 |
|---|---|
| **アプリケーション** | 自動運転車 / 工場・倉庫 / センサー / ロボット |
| **現場のデータセット** 🔴赤枠 | 3D/4Dデータ、センシングデータ、シンセティックデータ |
| デジタルツイン | |
| 産業OS | |
| AIモデル ／ シミュレータ | WFA(Cosmos)、Alpamayo、VLA(RoboTTT) ／ CUDA、Isaac・OMNIVERSE |
| GPU | |

**赤枠は「現場のデータセット」の層にだけ引かれている。** ここが自分たちの戦える場所であり、かつ金が動く場所だ、という主張。

GPU・シミュレータ・産業OSの層はNVIDIA/Google/Siemens/ソフトバンクといったプレイヤーが取りに来る。「桁が1桁足りない」「発電所から作らないといけない」世界。そこは張らない。

これ、フェーズ2レポートで整理した「技術スタック」と同じ地図なんだけど、**そこに『どこで勝つか』の線が引いてある**のが決定的に違った。自分の整理は地図で止まっていた。

#### 核心の対比：自動化 vs 自律化

| | 従来ロボット（自動化） | Unitree的アプローチ（自律化） |
|---|---|---|
| モーター | ハーモニックドライブ、1台数百万円 | 3万円クラスの汎用モーター |
| 精度の担保 | 高精度ハードで制御通りに動かす | 精度が低い前提で、学習で吸収 |
| 学習 | 実機を壊しながら歩行練習 | シミュレータ内で数百体を並列コピー |
| 想定外 | 人間が全パターンを事前に想定 | 物理法則を経験値として勝手に学習 |

シミュレータ上で悪路・坂道・ぬかるみを数百体並列で走らせて深層強化学習させ、そのソフトを実機に入れると「ある程度いきなり動く」。安いモーターで動く。

Sim-to-Real の話自体はフェーズ2で整理済みだった。でも、**「だからハーモニックドライブが要らなくなった」というコスト構造の帰結**まで一気につながって聞こえたのは初めてだった。中国のコスト優位は「EVサプライチェーン転用」だけじゃない。**学習で精度要件そのものを下げた**ことにもある。

#### 合成データ

自動運転で「10万kmに1回しか起きない事故シーン」のデータをどう集めるか、という長年の難問。「通常運転は偏差の真ん中でいくらでも取れるが、赤丸の部分（レアケース）はどうするのか」。生成AIによる合成データで、レアデータを水増しして超高速で並列に回せるようになった。NVIDIA Cosmosを使った際の学習量が「2000万時間 ≒ 人間の人生でいえば2300年分」というスケール感の話も出た。

#### 現場の痛み（写真つき）

- 指を挟んだ怪我の写真（アームが持ち上げた瞬間に予期せぬ方向へ動いた）
- オフィスの壁紙がロボットで削れている
- 四足がガラスにバシャーン・バーンとぶつかりながらテストが続き、ミーティングに集中できない
- 夏場、モーターが熱暴走してバタンと倒れる。**これはシミュレータでは絶対に分からない**

最後の一点が重い。「Sim-to-Realギャップ」を接触リッチ・変形物体の話だと思っていた。**熱**という次元があった。

#### ループ

現場からデータを取る → 合成データで水増し → シミュレータで回す → 実機に入れる → 失敗する → またデータを取る。

**このループを回し続ける会社が強くなる。** 初めは結果が出ない。でもずっとやっていたら効いてくる。この「セットで」の話がお客さんになかなか伝わらない、という苦労も込みで。

### 2. 松崎大輔さん（合同会社サイビット / scibit.ai）— 「身体を得たAIの育て方」

冒頭の問いが良かった。

> **フィジカルAIは社会の中で「何」になるのか？**

会場からは「エッセンシャルワーカー」「人を助ける存在」という答え。この問いが最後にもう一度戻ってくる構成。

#### 育てるプロセス

1. 目的・価値・責任を決める
2. ハードウェア設計（どんなセンサーが要るか、手足が要るか）
3. 仮想空間で育てる（強化学習、VLA）
4. 現実にデプロイ → **だいたい一発では動かない** → 回す

「すぐ現場に導入して終わり、ということはない」。

#### アーキテクチャの整理

> **知能はクラウドへ、反射はエッジへ**

高度な思考はエネルギーを食うのでクラウドの巨大な計算機で。身体を動かす瞬間の反射はローカルのエッジで。この2層の往復がフィジカルAIのシステム構成。

#### 実演：Unitree G1 の強化学習

シミュレータ自体をLLMに作らせ、物理シミュレータ上で評価関数を設計して反復。約50回の反復で成功。

**一番面白かった失敗談：**

うつ伏せから立ち上がる動作を学習させようとした。報酬関数を「高さ」にしたら、**最初に跳ね上がってマックススコアを取る**という報酬ハックが発生して、うまく学習しなかった。

評価関数には省エネ指数（エネルギー消費）やダメージへのペナルティも入れた、と。

#### そして本題

> 報酬関数の設計とは、**我々が何をよしとするかの設計**である。

我々が脳に持っている「何を大切にし、何を守り、何を許さないか」という倫理観を、報酬・コストという数値に**翻訳**する必要がある。この翻訳は不完全なので、そのロスをできるだけ縮めるのが人間側のテクニック。

そして、デジタル空間で評価が欠落していると、実世界で動かした時にそこが危険になる。

> **評価設計自体が世界を動かしていく。**

最終的に責任を取るのは人間なので、「誰に任せて、誰が止めて、誰が直すか」を最初に決めておく必要がある。検証済みの行動が集まったものが、新しいインフラになる——という話でクロージング。

### 3. 角田俊さん（個人事業主 / YouTube「KITの未来生活プロジェクト」）— 天井からロボットアームを生やす

180cm・8.5kg のロボットアームを買って、自宅の**天井から吊るした**話。

> チャンネル: [@developer-kit](https://www.youtube.com/@developer-kit) —「『IT技術×モノづくり』で訪れるかもしれない"未来の日常"を試作してみる」
> 経歴: 2020/4〜 (株)神戸デジタル・ラボ、2023/10〜 個人事業主

**先に書いておくと、この人がやっていることは自分がやろうとしていることとかなり近い。** 個人が、自腹で、ハードを買って、失敗しながら、YouTubeで公開する。ロールモデルが一人見つかった日でもある。

**動機:** アイアンマンで、ロボットアームがはんだ付けを手伝ってくれるシーン。あれをやりたい。でも5畳の部屋に180cmのアームを床置きすると邪魔だし、ブンブン振ったら危ない。じゃあ天井から吊るせばいい。

**実装:**
- 石膏ボードの上の野縁（木）に荷重を持たせる
- アルミフレームで抱き合わせ、天井から少しだけ出す形で固定
- 強度計算は機械設計をやっている弟に電話で相談（「素人なのでどう考えていいか教えてもらった」）
- 屋根裏がめちゃくちゃ狭く、腰を痛めながら点検口と通路を作るところから
- 耐久テスト = **自分（70kg）がぶら下がってみる**

**失敗:**
納品当日、チェーンブロックで吊り上げようとしたが、チェーンブロック自体の長さが最短でも50cmあって物理的に持ち上がらない。落下したら誰か足を骨折する。**その日は断念**。

**再挑戦:**
納品時の梱包発泡スチロールを使い、アルミフレームでジグを自作。四隅を引っ掛けてバランスを見ながら2人で吊り上げ。安全靴着用。完成。

これ、まさにOpen Lab的なコンテンツだと思った。**「断念した日」が入っているから物語になっている。**

### 4. 高所点検ロボット（伸縮ポール）

現場課題ドリブンの実装。

**課題:**
- 高所作業は労働災害が多い（電子メーカー時代の実感として）
- 安全対策コストが増える
- 脚立設置、高所作業者の手配 → **点検自体より準備に時間がかかる**
- 人手不足で点検の頻度を維持できない
- 対象は警報機・ダクトなど、天井付近の設備

**解決:**
移動体＋伸縮ポール。収納時20cm → 伸長時2m。ロボット本体から給電し、Raspberry Pi から制御。シリアル通信＋Pythonで伸ばして・下ろして・値を取得。

**「2週間前に買って、今ここまで動いている」**。他の仕事の片手間で1週間。

ノイズ対策（駆動系と制御系の電源分離）、現場振動対策といった、ソフト屋だけでは出てこない話。「機械もソフトも全部いるので、そこら辺が強み」。

**次のステップ:** カメラを載せて自律化。

## 今日学んだこと

1. **実物に触ることは、情報摂取ではなく別のチャンネル。** 質量・熱・音は動画に映らない。フェーズ2〜4で積み上げた文献ベースの理解に、今日ようやく身体が追いついた。

2. **「自動化 vs 自律化」は、Physical AI を語る一番シャープな軸かもしれない。** NVIDIAの用語戦略を批判的に検討したフェーズ2の結論より、この対比の方が現場に届く言葉だと思う。

3. **技術スタックの地図には、「どこで勝つか」の線を引かないと使えない。** 杉本さんの図は自分のフェーズ2の整理とほぼ同じ構造なのに、赤枠が一本入っているだけで意思決定の道具になっていた。地図と戦略は違う。

4. **Sim-to-Realギャップに「熱」がある。** 接触リッチ・変形物体だけを見ていた。夏場のモーター熱暴走はシミュレータに乗らない。これは自分のSO-101でも起きる話だ。

5. **報酬関数 = 倫理の翻訳。** 「Road to Folding a T-shirt」の報酬関数を、いつか自分が書く。その時、自分は「たためた」を何で測るのか。跳ね上がってスコアを取る G1 と同じことを、自分のロボットも必ずやる。

6. **失敗の日が入っているから物語になる。** 角田さんの「チェーンブロックが50cm長くて断念した日」が、あの発表を面白くしていた。これはプロジェクト憲法の裏付けだ。
